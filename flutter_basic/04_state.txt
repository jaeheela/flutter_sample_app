상태관리

1. 리스트화면
- 고양이 사진이 추가될 때마다 목록 갱신되기

2. 상세화면
- 좋아요가 해제된 상태에서는 좋아요가 설정되어야 함 (스위치 버튼)
- 좋아요가 설정된 상태에서는 좋아요가 해제되어야 함 (스위치 버튼)
- 좋아요 수가 1씩 증가될 때마다 좋아요 수가 1씩 증가해야 함
- 댓글이 추가될 때마다 댓글 수가 1씩 증가헤야 함
- 댓글이 작성되면 새로운 댓글이 댓글 목록에 추가되어야 함

3. 업로드화면
- 업로드한 고양이 사진이 표시되어야 함


<플러터의 상태관리 규칙>
- 변수의 값이 바뀔 때마다 화면에 반영되어야 하기 위해서는 해당 변수가 state 이어야 함
- 상태에 해당하는 변수에 값이 바뀔 때에는 화면이 업데이트 됨
- 상태가 아닌 변수의 값이 바뀔 때에는 화면이 업데이트 되지 않음
- 변수가 state가 되기 위한 조건
    : state를 상속받은 클래스 내부에서 선언하기
    : 변수의 값을 변경할 때 플러터에서 제공하는 방식 사용하기 : setState 함수
- setState 함수
    : 함수를 입력받는 함수임
    : setState의 입력으로 주어지는 함수에서 변수의 값을 변화시킴
